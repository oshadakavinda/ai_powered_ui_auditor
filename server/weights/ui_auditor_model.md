# UI/UX AI Auditor

An AI-powered tool that analyzes User Interfaces (UI) and evaluates their style and components against a library of expert-level web and mobile designs. By leveraging object detection and image vectorization, it assesses how closely a given UI design aligns with professional industry standards.

## Core Features

1. **UI Component Detection**: Uses a custom-trained **YOLOv8** model (`best.pt`) to detect and crop individual UI elements (buttons, text fields, images, etc.) from mobile and web screenshots.
2. **Expert Style Library Generation (AI Harvester)**: Automatically processes large datasets of mobile (Rico) and web (YashJain) UIs, categorizing and saving cropped components into an `Expert_Library`.
3. **Style Vectorization**: Utilizes OpenAI's **CLIP** model (`ViT-B/32`) to convert expert UI components into mathematical feature vectors.
4. **Fast Similarity Search**: Uses a **FAISS** vector database (`expert_style_index.bin`) to index expert style vectors for rapid retrieval.
5. **Automated Auditing**: Takes an input UI design, detects its components, vectorizes them, and compares them against the FAISS index to generate a visual "Audit Report" with similarity scores, bounding boxes, and an overall grade.

## File Structure

- `vectorDatabase.ipynb`: The main notebook containing the pipeline for harvesting UI data, building the FAISS vector database, and generating final audit reports.
- `detect_features.ipynb`: Utility notebook for basic component detection and batch crop extraction using YOLO models.
- `best.pt`: Custom trained YOLOv8 model for UI component detection.
- `yolov8n.pt`: Pre-trained YOLOv8 nano model for general object detection.
- `expert_style_index.bin`: The compiled FAISS vector index of expert UI styles.
- `app_details.csv` & `ui_details.csv`: Metadata used to map UI images to their respective industry categories.
- `merged_industry_map.csv`: Generated by merging the above CSVs to organize the Expert Library.
- `image_paths.txt`: A reference list mapping vectors in the FAISS index back to their original image paths.

## How It Works

The system operates in three main phases:

### Phase 1: Data Harvesting & Preparation
Extracts UI elements from thousands of professional mobile and web designs. The `AI Harvester` uses YOLOv8 to locate and crop UI components, saving them into a structured directory (`Expert_Library`) organized by category and platform.

### Phase 2: Vectorizing the "Brain"
The cropped expert elements are passed through the CLIP model to create deep-learning style vectors. These vectors are indexed into FAISS, creating a highly efficient searchable "Brain" of professional UI patterns.

### Phase 3: The Audit
When testing a new UI design (e.g., `test_ui.png`), the system:
1. Detects elements using YOLOv8.
2. Crops and vectorizes each element with CLIP.
3. Queries FAISS for the nearest neighbor match within the Expert Library.
4. Calculates a similarity percentage based on vector distance.
5. Generates an annotated image (`final_audit_report.jpg`) highlighting each element's score and an overall evaluation (e.g., "EXCELLENT", "GOOD", "NEEDS WORK").
